# Командная работа, Git, Kaiten

## Порядок работы

1. Получаем задачу от лида/выбираем самостоятельно.
2. Создаем новую ветку для конкретной задачи.
    - Ставим задаче статус "_В работе_".
3. Когда все готово создаем `pull request` в ветку `develop`.
    - Ставим задаче статус "_Пул реквест_".
4. Проводим командное ревью.
5. Если получили минимум 2 апрува от коллег:

    - Вливаем в ветку develop (через merge).
    - Ставим задаче статус "_Готово_".

6. Оповещаем всех остальных, чтобы они подтянули свежий `develop` и сразу порешали конфликты, если они есть.

## Ветвление проекта

1. `main` - основная ветка в которой хранится продакшн-версия приложения (вливаем код из `develop` после прохождения каждого спринта).

2. `develop` - ветка в которой ведется разработка. Стабильная версия приложения из этой ветки затем мерджится в `main`.

3. `{номер спринта.номер задачи}_{описание}` - например `5.14_add_documentation`, временная ветка у конкретного разработчика под конкретную фичу/задачу, которая по готовности вливается в `develop` через `pull request`.

## ❗️ 2 важных правила

**1. Не использовать `git rebase` в уже запушенных ветках (запушенных вами или другим разработчиком)**. Это может привести к существенным конфликтам в репозитории.

**2. Пушить только в свою ветку разработки под отдельную фичу. Не пушить напрямую в develop или main.**

## Как правильно устанавливать зависимости?

В проекте используется `monorepo` на основе [`lerna`](https://github.com/lerna/lerna)

Чтобы добавить зависимость для клиента:
`yarn lerna add {your_dep} --scope client`

Для сервера:
`yarn lerna add {your_dep} --scope server`

И для клиента и для сервера:

`yarn lerna add {your_dep}`

Если вы хотите добавить dev зависимость, проделайте то же самое, но с флагом `dev`
`yarn lerna add {your_dep} --dev --scope client`
